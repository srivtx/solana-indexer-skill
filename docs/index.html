The gap

Fills an unsolved idea from the AI Kit’s own Ideas table.

The Solana AI Kit’s Ideas table in solanabr/skills lists indexing-geyser — “Geyser plugins and data indexing” — as an unsolved gap with zero implementations.

No existing skill teaches how to build a Solana indexer. Helius, QuickNode, and solana-agent-kit all teach how to consume indexed data. Every serious Solana dApp — DeFi dashboards, NFT marketplaces, gaming leaderboards, social graphs, analytics — needs a custom indexer. Until now, every founder had to rediscover the same patterns.

The decision tree

Ask “I need to index X”. Get the right path.

indexer-architecture.md is the front door. Five viable ingestion methods, each with different tradeoffs on cost, latency, complexity, and what you can do.

Geyser gRPC

Rust · high-volume · <100ms

Custom filtering, on-prem data. Best for high-throughput indexers.

Enhanced WebSocket

TS / JS · most dApps · 200-500ms

Helius or QuickNode. The default choice for most teams.

Webhooks

Any lang · event-driven · 1-5s

Helius or QuickNode webhooks. Low-throughput, simple.

Polling

Any lang · cold start · 5-30s

getSignaturesForAddress. Simple backfill, expensive at scale.

Subgraph

GraphQL · public data · 30s-2m

The Graph on Solana. Queryable GraphQL, lowest complexity.
